# User/App API

Source: JADX-decompiled Divoom Android app, master command list at
`references/apk/decompiled_src/sources/com/divoom/Divoom/http/HttpCommand.java`
(absolute path, gitignored — not shipped in this repo). Request shapes from
`http/request/**/*.java` (`@JSONField(name=...)`), response shapes from
`http/response/**/*.java`, callers found by grepping the `HttpCommand.<Constant>`
name across `view/fragment/**` and `**/model/*.java`.

Public-doc cross-check: the community REvoom Team reference
(https://divoom.2a03.party/api/app.html, https://divoom.2a03.party/api/gz.html)
lists `User/ModifyCountryISOCode`, `User/PostCountryISOCode`, `User/SetAppVersion`,
`User/SetUserNewSign`, `User/ClickAdvert`, `User/DeleteUser`, `User/GetBackgroundImage`,
`App/DelUser`, `App/GetOtherUserList`, `App/SetIP` as real (but field-undocumented)
`appin.divoom-gz.com` endpoints — confirms existence, adds no field-level detail beyond
the APK. No public source documents request/response fields for any command in this
batch. All other commands below have no public documentation at all.

All endpoints other than `User/NewGuest` / `App/GetServerUTC` return the standard
`BaseResponseJson` envelope (`ReturnCode`, `ReturnMessage`) unless a richer response
class is noted.
