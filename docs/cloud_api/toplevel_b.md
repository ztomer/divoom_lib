# Top-level API (batch B)

Source: JADX-decompiled Divoom Android app, master command list at
`references/apk/decompiled_src/sources/com/divoom/Divoom/http/HttpCommand.java`
(absolute path, gitignored — not shipped in this repo). Request shapes from
`http/request/**/*.java` (`@JSONField(name=...)`), response shapes from
`http/response/**/*.java`, callers found by grepping the `HttpCommand.<Constant>`
name across `view/fragment/**` and `**/model/*.java`. These are top-level
commands (no `"Domain/"` prefix), so the grep was for the exact string value.

Public-doc cross-check: the community REvoom Team reference
(https://divoom.2a03.party/api/) documents `/ApplyBuddy`, `/ConfirmBuddy`,
`/Cloud/GalleryInfo`, `/Cloud/GetHotExpert`, `/Cloud/GetHotTag` and confirms
`UserLogin` (`Email` + MD5 `Password`, RC=5 for "password not match") as a
real `appin.divoom-gz.com` endpoint — consistent with what's already shipped
in `divoom_lib/divoom_auth.py`. No public source documents field-level shapes
for any other command in this batch; everything else below is decompiled-only.

All endpoints return the standard `BaseResponseJson` envelope (`ReturnCode`,
`ReturnMessage`) unless a richer response class is noted.
