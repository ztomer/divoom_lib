# Cloud/CloudPhoto/Community/Ali API

Reference catalog for the REST of the `Cloud/*` domain (upload + gallery social
features), plus `CloudPhoto/*` (device photo-frame albums), `Community/*`
(comment moderation), and `Ali/*` (Alibaba OSS direct-upload plumbing).

Source: decompiled Divoom Android app (JADX) at
`references/apk/decompiled_src/sources/com/divoom/Divoom/` — master command
list in `http/HttpCommand.java`, request shapes in `http/request/**`, response
shapes in `http/response/**`, callers found by grepping the `HttpCommand.*`
constant across `view/fragment/**` and `bean/cloud/PixelBean.java` →
`e3/g.java` (the actual upload implementation PixelBean delegates to).

A public (reverse-engineered) endpoint list exists at
[divoom.2a03.party/api/app.html](https://divoom.2a03.party/api/app.html)
(REvoom Team); it independently confirms `Cloud/GetHotTag` and lists
`Cloud/GalleryUploadV3` with the same field set derived below
(`Classify`, `Content`, `CopyrightFlag`, `FileMD5`, `FileName`, `FileSize`,
plus a `DeviceId`-like field). No public docs were found for the `Ali/*`
endpoints — they appear to be internal-only OSS plumbing.
