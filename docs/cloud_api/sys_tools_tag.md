# Sys/Tools/Tag/misc API

Source: JADX-decompiled Divoom Android app, `references/apk/decompiled_src/sources/com/divoom/Divoom/`
(master list `http/HttpCommand.java`; shapes from `http/request/**` and `http/response/**`
`@JSONField` annotations; callers cross-checked in `view/fragment/**` and `**/model/*.java`).
Public cross-reference: the community reverse-engineering doc at
`https://divoom.2a03.party/api/app.html` lists several of these same command strings
(`Sys/SetAPO`, `Sys/SetLogo`, `Sys/SetText`, `Sys/SetLightColor`, `Sys/SetLightFront`,
`Sys/SetLightBack`, `Sys/SetNotifySound`, `Sys/SetTextDirection`, `Test/SetUrl`,
`Tools/GetNoiseStatus`, `Log/SendLog`) as real (mostly undocumented) endpoints, with no field
detail — the field-level detail below comes from the decompiled request/response classes.

## Unknown / no signal

None. All 32 commands in this batch were matched to a `HttpCommand.java` constant and at least
a decompiled request or response class; most also had a confirmed live caller in `view/`. The
weakest-evidence entries — `Sys/SetAPO`, `Sys/SetLightBack`, `Sys/SetLightColor`,
`Sys/SetLightFront`, `Sys/SetNotifySound`, `Sys/SetTextDirection`, `Sys/SetText`, `Test/SetUrl` —
have real decompiled classes with concrete `@JSONField` names but no confirmed live UI call site
in this APK build (either superseded by the bulk `Sys/SetConf`/`SysDevUpdateConf` calls, gated
behind a hidden debug path, or simply not wired into a visible fragment); they are marked
`decompiled` rather than `unknown` because the wire shape itself is verified, and — for the
`Sys/Set*` ones — corroborated as real, live endpoints by the independent community
reverse-engineering doc at `divoom.2a03.party/api/app.html`.
