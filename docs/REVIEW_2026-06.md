## 1. Code & Architecture

### 1.1 `cmd_push_gif` calls `show_image` not `show_gif`
- **File**: `cli_commands.py:233`
- **Severity**: bug
- **Detail**: `cmd_push_gif` calls `show_image()`, which sends a static-image
  command. Animated GIFs are silently rendered as their first frame. Should call
  the GIF variant or detect frame count.

### 1.2 Dual notification monitoring
- **Files**: `gui_api.py:notifications_mixin`, `daemon.py:notification_monitor`
- **Severity**: consistency hazard
- **Detail**: Both the GUI and the daemon poll the same macOS notification SQLite
  DB on independent 1-second timers. Two processes tripping over each other =
  double-routed notifications, missed dedup, DB contention. Should be owned by
  exactly one process (the daemon) and forwarded to the GUI over MCP or a pipe.

### 1.3 Dual device access paths
- **Severity**: architecture hazard
- **Detail**: The GUI accesses the device both directly (`current_divoom`) and
  through the daemon (`daemon_client`). Half-migrated architecture: some features
  go through the daemon (MCP, exclusive mode), others reach the BLE socket
  directly. This is the root cause of consistency bugs when switching between
  modes. Finish the migration or eliminate one path.

### 1.4 `_Ring` class reinvents `collections.deque`
- **File**: `command_queue.py`
- **Severity**: medium (93 lines)
- **Detail**: `_Ring` is a fixed-size circular buffer — 93 lines, manual index
  arithmetic, `__repr__`, `__enter__`/`__exit__`. `collections.deque(maxlen=N)`
  is a C-optimised drop-in replacement with identical semantics, better
  performance, no custom memory management. Delete `_Ring`; use `deque`.

### 1.5 `DivoomGuiAPI` god object
- **File**: `gui_api.py`
- **Severity**: medium
- **Detail**: 454 lines, 12+ concerns (BLE, daemon lifecycle, notifications,
  weather, alarms, time, display, files, presets, factory reset, screenshot,
  MCP). `__dict__` passed to sub-APIs as mutable shared state — no locking, no
  ownership boundary. Split per concern (e.g., `DisplayAPI`, `DeviceAPI`,
  `DaemonAPI`, `FileAPI`) and inject only what each needs.

### 1.6 Async bridge blocks UI thread
- **File**: `gui_api.py:_run_async`
- **Severity**: medium
- **Detail**: `_run_async` creates a new event loop in a thread and calls
  `future.result()` — blocking the PyWebView JS bridge thread until the async
  op completes. For long BLE operations this freezes the UI. Switch to a
  fire-and-forget pattern with a callback that pushes results to JS when ready.

### 1.7 Race condition in daemon shutdown
- **File**: `daemon.py`
- **Severity**: medium
- **Detail**: After sending a keyboard interrupt, the shutdown path uses a 250ms
  `time.sleep` heuristic instead of waiting for the write-completion callback.
  Can corrupt the BLE socket state on rapid restart. Use a `threading.Event`
  that the write-completion path sets.

### 1.8 Deprecated asyncio APIs
- **File**: `mcp_server.py`
- **Severity**: medium
- **Detail**: Uses `loop=` keyword argument (removed in Python 3.14) and private
  `asyncio.streams.FlowControlMixin`. Both will break on Python 3.14+. Switch to
  the `asyncio.get_running_loop()` pattern and `asyncio.Protocol`.

### 1.9 `sys.path` manipulation
- **Files**: `cli_commands.py:cmd_daemon`, `cmd_menubar`
- **Severity**: low
- **Detail**: Injects the project root via `os.path.dirname(...)` and `sys.path.insert`.
  Prevents the project from being a proper pip installable package. Use
  `pip install -e .` + entry-points instead.

### 1.10 `build_parser()` in `cli.py`
- **Severity**: low
- **Detail**: Single 70-line function building all sub-parsers. Violates Open/Closed.
  Move each sub-command's argument setup to its handler module.

### 1.11 `inspect.iscoroutinefunction` dispatch
- **File**: `mcp_server.py`
- **Severity**: low
- **Detail**: The dispatch function inspects whether the handler target is a
  coroutine at every call. Since the set of coroutine handlers is static in the
  mapping, pre-compute the split at init time.

## 2. Design / UI

### 2.1 ~40 inline `style` attributes across templates
- **Files**: `templates_*.js` (settings, routines, channels, monthly_best)
- **Severity**: medium
- **Detail**: Every inline style is a design decision that cannot be themed,
  reused, or overridden without a JS change. Create CSS tokens — `.flex-row`,
  `.text-label-sm`, `.form-row`, `.panel-inset` — and move styling to stylesheets.
  **Exception**: `colspan="2"` and trivial widths that genuinely vary per instance
  may stay inline as part of the structural template.

### 2.2 Conflicting active-state conventions
- **Files**: `tabs.css`, `style_extra.css`
- **Severity**: low
- **Detail**: Tab active states use `background: var(--primary)` (solid orange),
  selector cells and some card states use `rgba(0, 255, 204, 0.12)` (green tint).
  Two different visual languages for "this thing is selected". Pick one.

### 2.3 `!important` sprawl in `appbar.css`
- **File**: `appbar.css`
- **Severity**: low
- **Detail**: 6 `!important` flags on `#global-status-dot` selectors. Each one
  bypasses the cascade and makes future overrides harder. Fix the specificity
  chain instead.

### 2.4 Dead CSS
- **Files**: Various `.css` files
- **Severity**: low
- **Detail**: `.color-picker-grid`, `.color-swatch.*`, `.range-slider`,
  `.spinner`, `.channel-grid` — all defined but unused. Remove dead code.

### 2.5 Presentation logic in JS strings
- **Files**: `templates_*.js`
- **Severity**: medium
- **Detail**: Alarm rows, the arranger popup, image controls, and several other
  UI sections build HTML strings with inline styles. Use `<template>` elements
  (in `index.html` or loaded via JS) and clone them — the browser's HTML parser
  is faster and catches syntax errors at page load rather than at interaction time.

## 4. Architecture (Cross-Cutting)

### 4.1 Daemon ↔ GUI responsibility split
The half-migrated daemon architecture is the single biggest risk. The review
identified at least five symptoms of it (dual notification monitoring, dual
device access, MCP/hardware-mode inconsistency, race on shutdown, race on
startup). A clear "daemon owns the device" boundary — with the GUI talking to
the daemon exclusively over the MCP pipe in all modes — would eliminate all five.

### 4.2 Event/notification bus
The project has at least three notification mechanisms (macOS SQLite polling,
MCP response polling, direct BLE notification callbacks) with no central dispatch.
A lightweight pub/sub bus (even an in-process `threading.Event` + callback dict)
would let any component emit an event and any other subscribe without coupling.

### 4.3 Inline style → CSS token migration
The ~40 inline styles are a design-debt item that compounds every time someone
adds a new card or row. A one-time pass to define the common patterns as CSS
classes (`.flex-row`, `.toggle-bar`, `.panel-inset`, `.text-label-sm`) and
replace the inline styles would pay for itself on the next UI change.
